## Casos prováveis de falha

A partir daqui, casos onde sei pela leitura do código que o conversor não cobre. Confirmar exactamente como cada um sai no `.tex` e no PDF do Overleaf.

**(a) Cifrão literal misturado com matemática.** O salário é \$5 e a fórmula é $x = 5$. O regex de math é não-greedy — provável que capture incorrectamente.

**(b) Imagem.** ![alt text](https://example.com/img.png) — o `!` inicial cai fora do regex de link; provável que o link interno seja capturado, com `!` literal antes.

**(c) Tabela Markdown.**

| col1 | col2 |
|------|------|
| a    | b    |

Tabelas não estão no conversor; o esperado é texto plano escapado.

**(d) HTML inline.** <strong>bold via HTML</strong> — não suportado, esperado escapes literais (`\textless{}strong\textgreater{}`).

**(e) Link com parênteses no URL.** [foo](https://example.com/path(with)parens) — o regex `\(([^)]+)\)` não lida com parênteses aninhados.

**(f) LaTeX puro fora de math.** \textbf{bold via LaTeX} — esperado escape literal.

**(g) Lista aninhada.**

- parent
  - child
  - outro child
- segundo parent

Aninhamento por indentação não está implementado.

**(h) Mistura de listas adjacentes.**

- bullet
1. ordered
